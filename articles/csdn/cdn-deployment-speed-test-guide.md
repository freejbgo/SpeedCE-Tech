## 写在前面：上了 CDN，反而有人打不开了？

CDN 的本意是让网站更快、更稳。但现实里，接入 CDN 后的第一周，往往是工单最多的一周：

- 「广东快了，新疆更慢了」
- 「直连源站没事，走 CDN 就 502」
- 「HTTPS 证书报错，但源站证书明明没过期」
- 「开发说回源正常，运营说用户打不开」

问题通常不在「CDN 有没有开」，而在 **验收方法不对**。

很多人只在自己电脑刷新两下首页，就认为 CDN 接入成功。这只能证明 **你本地 → 最近边缘节点 → 源站** 这一条路径没问题，证明不了全国用户。

本文是一份 **CDN 全生命周期多节点测速指南**：从选型对比、切量前验收、切量中监控到故障排查，每一步都说明该测什么、怎么读地图、什么时候该找 CDN 客服、什么时候该查源站。操作示例使用 SpeedCE，但方法论适用于任何多节点拨测工具。

## 第一章：CDN 测速和「网站测速」不是一回事

### 1.1 三个层次，别混着测

| 层次 | 测什么 | 典型工具/方式 |
|------|--------|----------------|
| 源站层 | 机器、Nginx、证书、安全组 | SpeedCE 测源站 IP 或 origin 域名 |
| CDN 层 | 边缘节点、缓存、回源、CNAME | SpeedCE 测 CDN 加速域名 |
| 页面层 | JS/CSS 体积、渲染性能 | PageSpeed、WebPageTest |

**SpeedCE 负责前两层**：回答「从各地看，CDN 域名和源站能不能访问、延迟如何」。

### 1.2 必须建立的「对照思维」

CDN 排障的黄金法则：

> **同一时刻，用相同协议，分别测 CDN 域名和源站，对比两张地图。**

| CDN 域名 | 源站 | 结论 |
|----------|------|------|
| 红 | 绿 | CDN 配置/节点/证书问题 → 找 CDN |
| 绿 | 红 | 源站挂了，CDN 只是「背锅」 |
| 红 | 红 | 先修源站 |
| 绿 | 绿 | 基础设施正常，查缓存策略或个别边缘节点 |

没有对照，你只能猜。

## 第二章：SpeedCE 测 CDN 的标准操作

### 2.1 测前准备

确认你有以下信息：

| 项目 | 示例 | 用途 |
|------|------|------|
| CDN 加速域名 | `www.example.com` | A 组测试目标 |
| 源站地址 | `origin.example.com` 或 `1.2.3.4` | B 组测试目标 |
| 协议 | HTTPS（生产环境首选） | 与真实用户一致 |
| 业务区域 | 国内 / 全球 | 选中国节点或全球节点 |

### 2.2 标准四步

1. 打开 https://speedce.com/?lang=zh-CN
2. 选择 **HTTPS** + **中国节点**（国内业务）或 **全球节点**（出海）
3. 输入 CDN 域名 → 开始测速 → 截图保存（**地图 A**）
4. 输入源站 IP 或源站域名 → 再测 → 截图保存（**地图 B**）

若源站域名未对公网暴露，只测 IP 即可。

### 2.3 三网分离（国内 CDN 必做）

在地图 A 上分别筛选 **电信 / 联通 / 移动**，重复地图 B 对照。

CDN 厂商常说「三大运营商全覆盖」，但边缘节点质量常有差异——**移动用户投诉最多**，移动地图要单独看。

## 第三章：切量前——CDN 上线验收清单

在把 DNS CNAME 切到 CDN **之前**，用下面清单逐项验收。

### 3.1 源站先行

- [ ] 源站 IP + HTTPS：中国节点通畅率 ≥ 95%
- [ ] 源站证书有效，SAN 包含所有加速域名
- [ ] 源站已配置 CDN 回源 Host（避免回源 404）
- [ ] 源站防火墙已放行 CDN 回源 IP 段（厂商文档有列表）

**源站不通，开 CDN 没意义。**

### 3.2 CDN 控制台配置

- [ ] 加速域名已添加，CNAME 已分配
- [ ] HTTPS 证书已上传或使用 CDN 免费证书
- [ ] 强制 HTTPS 跳转已按需开启
- [ ] 缓存规则：静态资源长缓存，API/HTML 按业务设置
- [ ] 回源协议与源站一致（HTTP 回源 / HTTPS 回源）

### 3.3 切量前「hosts 模拟」+ SpeedCE

在切 DNS 之前，可在本地 hosts 绑定 CDN 节点 IP 做预览。预览通过后：

1. SpeedCE 测 CDN 域名（若已能通过 hosts 或测试子域访问）
2. 与源站地图对照

### 3.4 切量前检查表（可打印）

```
□ 源站 HTTPS 地图全绿
□ CDN 测试域名 HTTPS 地图全绿
□ 电信/联通/移动三张 CDN 地图无明显短板
□ HTTP → HTTPS 跳转正常
□ 关键子域（api / static / m）已单独添加 CDN 并测过
□ 回源 5xx 在 CDN 控制台无告警
□ 地图截图已存档
```

## 第四章：切量中——DNS 切换的 72 小时怎么测

### 4.1 为什么切量期要「密集点检」

DNS TTL 导致各地生效时间不同。你可能已切了 2 小时，新疆用户仍解析到旧 IP。

**SpeedCE 的价值**：看异常点是「随机消散」还是「固定省份持续红」。

### 4.2 推荐测速节奏

| 时间点 | 操作 |
|--------|------|
| 切 DNS 前 | 源站 + CDN 各测 1 次，存档 |
| 切 DNS 后 10 分钟 | CDN 域名测 1 次 |
| 30 分钟 | 再测 1 次 |
| 2 小时 | 再测 1 次 |
| 24 小时 | 三网分离各测 1 次 |
| 72 小时 | 最终验收测 1 次 |

### 4.3 地图形态解读

| 现象 | 含义 | 处理 |
|------|------|------|
| 异常点随时间减少 | DNS 缓存刷新中，正常 | 继续等待 |
| 固定某省持续红 | 该省 DNS 缓存顽固或区域节点故障 | 联系 CDN 查该省节点 |
| 全国同时红后恢复 | 可能切量瞬间源站过载 | 查源站 CPU/带宽、回源并发 |
| 全国持续红 | CDN 配置或源站故障 | 对照源站地图 |

### 4.4 迁 DNS 时的 TTL 建议

切量前 24–48 小时，把 DNS TTL 调到 **300 秒（5 分钟）**。切量后恢复为 3600 或更高。

SpeedCE 地图 + 低 TTL，能把「用户说我这边还不行」的扯皮，变成「地图显示你省已全绿，请清本地 DNS 缓存」。

## 第五章：六大 CDN 故障场景——对号入座

### 场景 1：CDN 绿、源站红

**现象**：用户访问 CDN 域名正常，直连源站 IP 失败；或 CDN 控制台回源失败率飙升。

**SpeedCE 测法**：

- A：HTTPS + 中国节点 + CDN 域名
- B：HTTPS + 中国节点 + 源站 IP

**判断**：B 红 A 绿 → 源站问题（进程挂了、证书过期、安全组拦了 CDN 回源 IP）。

**处理**：修源站；检查 CDN 回源 IP 白名单。

---

### 场景 2：CDN 红、源站绿

**现象**：源站直连正常，走 CDN 域名打不开或极慢。

**可能原因**：

- CDN 证书未配或过期
- CNAME 未生效、配错
- CDN 边缘节点故障
- 缓存了错误页面
- 部分省份节点回源超时

**SpeedCE 测法**：对照 A/B 地图；三网分离看是否单网问题。

**处理**：

1. CDN 控制台查证书、CNAME、节点状态
2. 提交工单附 SpeedCE 地图截图（注明异常省份）
3. 尝试「刷新全站缓存」后复测

---

### 场景 3：全国绿，但个别省份慢

**现象**：通畅率 100%，但某省平均延迟明显高于其他省。

**SpeedCE 测法**：看地图颜色虽绿，对比数字面板平均延迟；分省看异常高亮区域。

**判断**：CDN 边缘调度问题——该省用户被分配到了较远节点。

**处理**：CDN 控制台查看「分区域回源」或「智能解析」策略；工单要求优化该省节点覆盖。

---

### 场景 4：静态资源绿、API 红

**现象**：首页能开，登录/下单失败。

**SpeedCE 测法**：

- 测 `www.example.com`
- 测 `api.example.com`（若独立子域）

**判断**：可能只给 www 加了 CDN，api 未配；或 api 回源超时设置过短。

**处理**：子域分别配置 CDN；API 路径设短缓存或回源。

---

### 场景 5：HTTPS 报错，HTTP 正常

**现象**：用户报证书错误；SpeedCE HTTPS 红、HTTP 绿。

**可能原因**：

- CDN 证书过期
- 证书只签了 www，未签根域或子域
- SNI 配置错误
- 混合内容（页面 HTTPS、资源 HTTP）——SpeedCE 不测页面内容，需浏览器开发者工具

**处理**：CDN 控制台更新证书；用 SpeedCE HTTPS 复测至全绿。

---

### 场景 6：海外快、国内慢（或相反）

**现象**：出海业务，两端体验割裂。

**SpeedCE 测法**：

1. 全球节点 + CDN 域名
2. 中国节点 + CDN 域名
3. 与源站对照

**解读**：

| 中国 | 全球 | 建议 |
|------|------|------|
| 慢 | 快 | 考虑国内 CDN 或国内加速包 |
| 快 | 慢 | 检查海外节点覆盖，或加海外 CDN |
| 都慢 | — | 源站带宽/性能瓶颈 |

## 第六章：CDN 选型阶段——如何用测速做决策

### 6.1 试用期对比法

多数 CDN 提供免费额度或试用。同一源站、同一域名（用不同测试子域），分别接入 A/B 两家 CDN：

| 对比项 | CDN A | CDN B |
|--------|-------|-------|
| 中国节点通畅率 | | |
| 电信平均延迟 | | |
| 移动平均延迟 | | |
| 全球目标国延迟 | | |
| 控制台易用性 | | |
| 回源失败率 | | |

**SpeedCE 负责填前五行**——比看广告文案靠谱。

### 6.2 免费 CDN vs 付费 CDN

| 类型 | 测速关注 |
|------|----------|
| Cloudflare 免费版 | 全球节点好，国内可能慢——中国地图单独看 |
| 国内云厂商 CDN | 三网覆盖、备案要求、回源带宽价 |
| 第三方融合 CDN | 对照源站，防「假加速」 |

没有「最好」的 CDN，只有「你的用户在哪、地图谁更绿」。

### 6.3 与 PageSpeed 分工

- SpeedCE：CDN **有没有让各地能访问、延迟如何**
- PageSpeed：CDN **有没有让 LCP、FCP 达标**

两个都绿，才算 CDN 接入成功。

## 第七章：进阶技巧

### 7.1 「回源旁路」验证

临时关闭 CDN 代理（灰云 / 暂停加速），只用 SpeedCE 测源站，确认基线。再开启，测 CDN。避免把源站问题和 CDN 问题搅在一起。

### 7.2 子域清单法

```
www.example.com      → 页面 CDN
static.example.com   → 静态资源 CDN
api.example.com      →  API（可能不回 CDN）
origin.example.com   → 源站（仅内网或白名单）
```

每月用 SpeedCE 对前三个跑一遍 HTTPS + 中国节点。

### 7.3 工单沟通模板

```
【问题】CDN 加速域名 www.example.com 在新疆、甘肃多节点异常
【时间】2026-06-24 21:30
【协议】HTTPS
【对照】源站 IP x.x.x.x 同期全国通畅率 98%
【附件】SpeedCE 电信/移动地图截图各 1 张
【请求】请排查新疆、甘肃边缘节点回源是否超时
```

客服回复速度通常比「网站打不开帮忙看看」快十倍。

### 7.4 与监控体系配合

| 工具 | 角色 |
|------|------|
| SpeedCE | 切量期密集点检、故障第一现场 |
| CDN 控制台监控 | 回源率、5xx、带宽、QPS |
| UptimeRobot 等 | 7×24 可用率告警 |
| 日志分析 | 追溯具体 URL 失败 |

SpeedCE 是「地图快照」，不是长期监控——但故障发生后的 **第一张图** 应该来自它。

## 第八章：常见误区

### 误区 1：「CDN 开了就一定更快」

错。源站本身很慢、缓存未命中、回源跨洋，CDN 可能更慢。对照源站地图说话。

### 误区 2：「全国绿就不用看了」

通畅率 100% 不代表延迟均匀。要看数字、要看三网、要看目标省份。

### 误区 3：「证书在源站配了就行」

CDN 边缘也要证书。源站证书好、CDN 证书过期，用户照样报错。

### 误区 4：「只测首页」

API、静态子域、跳转域都要测。首页绿、接口红，业务照样挂。

### 误区 5：「测一次就验收」

切量后至少测 72 小时；晚高峰测一次。CDN 问题常在高峰期暴露。
