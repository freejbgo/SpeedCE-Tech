## 第一章：问题为什么难排查

单点测速有「地理偏见」：你在公司 Wi-Fi 上测一次，只能代表那条线路。多节点测速把「我一个人觉得快」变成「全国各地分别体验如何」。SpeedCE 用中国节点地图与全球节点地图呈现结果，通畅、异常、检测中、等待四种状态一目了然。

### 1.1 三个原则

| 原则 | 说明 |
|------|------|
| 对照测 | CDN 域名与源站、迁机前后、改配置前后，各测一张图 |
| 三网分 | 电信、联通、移动分开看，移动用户占比最高 |
| 多次测 | 间歇故障、DNS 生效、晚高峰，至少测 2–3 次 |

### 典型场景与 SpeedCE 测法
#### 场景 1：大促流量打满
**现象**：用户或业务方反馈与此相关的问题。
**SpeedCE 测法**：
1. 协议：**HTTPS**（按需切换 HTTP/PING）
2. 范围：**中国节点**
3. 输入目标域名或 IP → 开始测速

**地图怎么读**：
| 地图形态 | 可能原因 | 下一步 |
|----------|----------|--------|
| 全国大面积红 | 源站/安全组/证书全局故障 | 先修服务器，别怪用户网络 |
| 单省或单区域红 | 区域 DNS、线路或 CDN 节点问题 | 对照源站地图，定位 CDN 还是线路 |

| 全国绿但用户仍投诉 | 应用层、缓存、拦截问题 | 查日志；合规用 BOCE 等 |

## 第五章：常见误区

- **误区 1**：只在自己电脑测一次就下结论 → 必须多节点。
- **误区 2**：Ping 不通就认定线路差 → 可能禁 Ping，改 HTTPS。
- **误区 3**：全国绿等于没问题 → 看延迟、三网、目标省份。
- **误区 4**：测速替代监控 → 拨测是快照，长期监控另配。
- **误区 5**：网络绿等于应用绿 → CORS、业务逻辑另查。
